# PRD.md

## 1. 产品定位
Manus-lite 是一个单用户、本地优先的会话式 AI 客户端：
- 后端：Python + FastAPI，负责会话、消息、Provider 调度、记忆摘要、SSE
- 前端：Vue3 + Vite，负责会话 UI、流式展示、Think 展示、本地持久化

目标是以最小复杂度实现接近主流 AI 聊天产品的核心体验：**会话隔离 + 流式可见 + 记忆连续**。

## 2. 目标用户
- 个人开发者
- AI 应用学习者
- 需要本地 Ollama 优先、并可扩展云模型调用的用户

## 3. In Scope（当前范围）
1. 左侧会话列表与历史查看（支持删除）
2. 草稿态（刷新默认空白、`+ New` 进入草稿）
3. 首发消息后创建 pending 会话，首条流内容到达后再展示历史项
4. 标题规则：首问前 20 字（标准化后截断）
5. SSE 流式输出（start/thinking/delta/end/error）
6. 多会话并发隔离与会话级 loading
7. Think 独立展示（滚动、折叠、弹窗）
8. Markdown 渲染（正文 + think）
9. IndexedDB 持久化（会话/消息/think）
10. 后端记忆机制（window + summary snapshot）
11. Provider 抽象：Ollama / GLM / Codex
12. 删除确认弹窗（Element Plus，暗色风格统一）

## 4. Out of Scope（明确不做）
- 多用户登录与权限系统
- 文件上传/多模态
- 计费与配额系统
- 企业级审计合规模块
- 复杂 APM/Tracing 平台

## 5. 用户故事与验收标准

### Story A：我希望从空白草稿快速开始新对话
- Given 我刚刷新页面或点击 `+ New`
- When 我尚未发送第一条消息
- Then 右侧保持空白草稿态，输入框居中，左侧不新增历史项

验收：
- 默认不自动打开历史会话
- 草稿态输入框位置正确

### Story B：我希望首发后再展示历史会话，避免空会话污染
- Given 我在草稿态发送首问
- When 后端创建会话且流式开始
- Then 仅在收到第一条流内容后，左侧才显示该会话

验收：
- 无首条流内容时不显示该会话
- 会话标题为首问前 20 字

### Story C：我希望多会话并发不互相影响
- Given 会话 A 正在流式
- When 我切到 B 并继续发送
- Then A/B 各自流式与 loading 独立

验收：
- 不串流
- 回切 A 可继续看到增量

### Story D：我希望看到并管理模型 thinking 内容
- Given 模型返回 thinking 内容
- When 我查看当前会话
- Then think 在独立区域滚动展示，结束后折叠，可弹窗查看

验收：
- think 与正文分离
- 两者都可 Markdown 渲染

### Story E：我希望安全删除历史会话
- Given 我悬停某会话标题
- When 我点击删除并确认
- Then 会话从列表与本地持久化移除，后端标记归档

验收：
- 删除按钮仅 hover/active 可见
- 弹窗风格与暗色主题一致
- 删除当前会话后回草稿态

## 6. 成功标准

### 功能指标
- 可稳定完成会话创建、聊天、流式、历史查看、删除
- 多会话并发下无串流/错写
- loading 状态与真实流状态一致

### 体验指标
- 首 token 可见时间可接受（本地模型条件）
- 滚动行为稳定（仅左栏与消息区滚动）
- Think 展示清晰且不干扰正文阅读

## 7. 失败与边界场景
1. Provider 不可用：`/health` 显示 false，聊天接口返回标准错误
2. 网络/CORS 错误：前端请求失败但界面不崩溃
3. 会话不存在：返回 404
4. SSE 中断：会话进入 error/done，不自动续传
5. 流式首次失败：pending 会话不进入历史列表

## 8. 当前状态
- 后端主链路可用：会话、消息、流式、摘要、provider 调用记录
- 前端主链路可用：草稿态、并发流式隔离、Think 展示、IndexedDB 持久化、删除确认流程
- 下一阶段聚焦：自动化测试覆盖与部署稳定性
