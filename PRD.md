# PRD.md

## 1. 产品定位
Manus-lite 是一个单用户本地优先的会话式 AI 客户端（前后端分离）：
- 后端：Python + FastAPI，负责会话管理、模型调用、SSE 流式输出、记忆摘要
- 前端：Vue3 + Vite，负责多会话 UI、流式展示、Think 展示和本地持久化

目标是提供接近主流 AI 会话产品的体验，强调“多会话隔离 + 流式可观测”。

## 2. 目标用户
- 个人开发者
- AI 应用学习者
- 需要本地模型优先（Ollama）且保留后续云模型切换能力的用户

## 3. In Scope（当前范围）
1. 会话列表与历史会话查看（支持删除）
2. 新会话草稿态与会话切换
3. SSE 流式输出（`/api/v1/chat/stream`）
4. 多会话并发隔离（会话 A 流式时，切到 B 不影响 A）
5. 会话级 loading 状态（左侧列表标记）
6. Think 内容展示（固定区域、折叠、弹窗）
7. Markdown 渲染（消息正文 + think 文本）
8. IndexedDB 持久化（会话、消息、think 状态）
9. 后端记忆机制（window + summary snapshot）
10. 模型接入：Ollama / GLM / Codex（统一 provider 抽象）
11. 删除确认弹窗采用 Element Plus，且视觉风格与暗色主题统一
12. 新会话历史项延迟展示：收到第一条流式内容后才进入左侧历史列表
13. 会话标题生成规则：首问文本标准化后截取前 20 字

## 4. Out of Scope（明确不做）
- 多用户登录与权限系统
- 文件上传与多模态
- 生产级可观测平台（APM、Tracing 平台）
- 复杂计费/配额系统
- 企业级审计与合规模块

## 5. 核心用户故事与验收

### 用户故事 A：我希望创建并持续进行对话
- Given 我打开页面
- When 我发送第一条消息
- Then 系统创建会话并返回流式回复；历史项在收到第一条流式内容后才显示

验收：
- 能看到回复增量输出
- 回复结束后消息落地到历史
- 左栏新会话标题为首问前 20 字

### 用户故事 A-1：我希望 New 时先进入干净草稿态
- Given 我点击 `+ New`
- When 尚未发送第一条消息
- Then 右侧聊天区保持空白，输入框垂直居中，左侧不新增历史项

验收：
- 刷新页面默认也是草稿态
- 草稿态输入框居中；进入已存在会话后恢复底部输入

### 用户故事 B：我希望多会话互不影响
- Given 会话 A 正在流式输出
- When 我切换到会话 B 并发送消息
- Then A 与 B 可并发流式，互不覆盖

验收：
- 左栏 A/B 可独立显示 loading
- 回切 A 时可继续看到增量更新

### 用户故事 E：我希望安全地删除历史会话
- Given 我在左侧历史会话标题区域悬停
- When 我点击删除按钮并确认
- Then 会话从左侧列表移除，同时清理本地持久化数据

验收：
- 删除按钮仅在 hover（或 active）时显示
- 删除确认弹窗为 Element Plus 样式且与暗色主题一致
- 删除当前激活会话后，右侧回到草稿态

### 用户故事 C：我希望长会话仍保留上下文
- Given 我持续多轮会话
- When 达到摘要阈值
- Then 后端生成记忆摘要并用于后续上下文构建

验收：
- `memory_snapshots` 有新增记录
- 后续回答上下文连续

### 用户故事 D：我希望刷新后仍能看到历史
- Given 我已有会话和消息
- When 我刷新页面
- Then 可恢复会话与历史记录（前端 IndexedDB + 后端接口）

验收：
- 会话列表仍在
- 历史消息可再次查看

## 6. 成功标准
### 功能可用指标
- 100% 支持创建会话、发消息、查历史、流式回复
- 多会话并发不串流
- 会话级 loading 与真实流状态一致

### 体验指标
- 首 token 可见时间在可接受范围（本地模型条件下）
- UI 滚动行为稳定（左侧列表、右侧消息区）
- Think 内容可查看、可折叠、可放大

## 7. 失败/边界场景
1. 模型不可用：健康检查显示 provider 不可用，聊天接口返回错误
2. 网络中断：SSE 结束于 error 事件，当前会话流状态结束并提示失败
3. CORS 配置错误：前端无法 fetch（当前已在后端允许 5173 源）
4. 会话不存在：返回 404
5. 流式中断：当前版本不做自动续传，允许用户重发
